## Definitions

### Level 2 document graph (`L2-DA-DOCUMENT-GRAPH`)

The directed graph whose nodes are normative Level 2 specification documents and whose edges are explicit document imports. The graph governs authority visibility only and does not represent workflow execution order or runtime data flow.

### Specification-document import (`L2-DA-IMPORT`)

An explicit same-level visibility edge from one Level 2 document to another Level 2 document named in the importing document's document.imports array.

### Local identifier (`L2-DA-LOCAL-IDENTIFIER`)

A definition, requirement, or relationship identifier declared directly by the document being validated.

### Transitive import closure (`L2-DA-IMPORT-CLOSURE`)

The importing document together with every document reachable by following zero or more explicit import edges.

### Visible identifier (`L2-DA-VISIBLE-IDENTIFIER`)

An identifier declared locally or by a document in the importing document's transitive import closure.

### Deterministic resolution (`L2-DA-DETERMINISTIC-RESOLUTION`)

The process of resolving a reference to exactly one globally unique visible identifier without precedence, shadowing, implicit imports, or search-order dependence.

### Document-authority conflict (`L2-DA-CONFLICT`)

Any duplicate specification identity, duplicate semantic identifier, ambiguous ownership, invalid parentage, unresolved root, unresolved import, import cycle, unresolved visible reference, or unresolved relationship endpoint.

### Relationship-endpoint visibility (`L2-DA-RELATIONSHIP-VISIBILITY`)

The requirement that each relationship source and target resolve to a visible definition or requirement in the declaring document's import closure.

### Import and runtime dependency distinction (`L2-DA-RUNTIME-DEPENDENCY-DISTINCTION`)

The normative separation between document imports, which grant specification identifier visibility, and workflow dependencies or runtime handoffs, which govern execution or data transfer and are outside this document's authority.
